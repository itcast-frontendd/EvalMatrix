# 2. The Law (Evaluation Dimensions)

You MUST follow this scoring procedure:

1. For each dimension, READ the band descriptors from **[10] → [7-9] → [4-6] → [1-3]**.
2. IDENTIFY which band the performance BEST matches — this is the "home band".
3. ASSIGN a score ONLY within that home band's range.
4. NEVER cross band boundaries: if performance fits [7-9], you MUST NOT score 6 or 10. If performance fits [4-6], you MUST NOT score 7 or 3.
5. Within a band, use the upper end if performance is strong for that band, lower end if borderline.
6. Then apply the **Hard Rules / Floors / Ceilings** below.  
   - If a hard rule conflicts with the initial band choice, the hard rule overrides.
7. Do **not** compensate one dimension with another.

8. For each dimension:
- If type is scale: output an integer score from 1 to 10.
- If type is binary: output true/false.
- If type is categorical: output ONE option exactly from the list.

# 4. Adjudication Protocol (Step-by-Step)
For EACH entry in the data block, execute the following mental process before generating the JSON:

1. **Context Lock**
   - Read Global Context first.
   - Identify speaker roles, entities, ongoing topic, discourse stage.

2. **Current Segment Grounding**
   - Read REFERENCE_TEXT if available.
   - Compare ASR with Reference; note whether ASR has errors.

3. **Output Inspection**
   - Check whether ANSWER preserves the segment’s meaning.
   - Check KERR, omission, addition, hallucination, tone, and segmentation.

4. **Streaming Continuity Check**
   - Use context_before / context_after to determine whether the answer is connectable and coherent across segments.
   - Penalize only material segmentation damage, not harmless restructuring.

5. **Band Selection**
   - For each dimension, choose the home band first.
   - Then apply Hard Rules.

6. **Concise Verdict**
   - Write reasoning in Simplified Chinese.
   - Keep it brief, specific, evidence-based.

     - “KERR错误：总经理→部長”

## 一、演示环境准备

### 1.1 快速启动

**Windows:**
```cmd
cd D:\POPO\AItester0227\AItester0227 - latest
双击 start.bat
```

**macOS / Linux:**
```bash
cd AItester0227
./start.sh
```

### 1.2 访问地址

| 服务 | 地址 | 说明 |
|-----|------|------|
| 前端界面 | http://localhost:3000 | 主要演示入口 |
| 后端 API | http://localhost:8000 | 数据接口 |
| API 文档 | http://localhost:8000/docs | Swagger 文档 |

### 1.3 演示前必做配置

1. 打开「模型管理」页面 (http://localhost:3000/config)
2. 配置 Judge 模型：
   - API Key: [你的 API Key]
   - Model: gemini-2.5-flash 或其他
   - API URL: https://aigw.netease.com/v1/chat/completions
3. 添加被测产品（至少 2 个用于对比）：
   - 方式 1：粘贴 OpenAI 兼容代码片段自动解析
   - 方式 2：手动填写 API URL 和 Key

## 二、演示流程（建议 8-10 分钟）

### 【第 1 幕】开场：产品定位与价值（1 分钟）

**画面：** 登录后主界面，展示整体布局

**讲解词：**
> "AI 是科技企业的生命线，而科学、高效的产品评测则是创造优秀 AI 产品的基石。今天为大家介绍 EvalMatrix —— 一套面向 AI 全场景的自动化智能评测平台。"

**展示要点：**
- 左侧导航栏：工作台、量化面板、人工复核、模型管理
- 三大核心优势：拒绝拍脑袋、即插即用、端到端自动化
- 整体布局简洁专业，功能分区清晰

**关键截图位置：**
1. 主界面全貌（展示导航栏和工作台）
2. 模型管理页面（展示已配置的产品列表）

---

### 【第 2 幕】核心功能：Judge Only 裁判模式（3 分钟）

**场景：** 已有模型输出，需要批量评分（最常用场景）

#### 步骤 1：准备测试数据（30秒）

**展示 Excel 文件格式：**

| 问题 | 模型A回答 | 模型B回答 |
|-----|----------|----------|
| 如何退换货？ | 您可以在订单页面申请退换货... | 请拨打客服电话... |
| 这个商品有优惠吗？ | 当前商品正在参加满减活动... | 没有优惠 |
| 物流多久能到？ | 一般 3-5 个工作日送达... | 很快 |

**讲解词：**
> "我们支持 Excel、JSON、JSONL、ZIP 多种格式。只需准备包含问题和模型回答的数据文件即可。"

#### 步骤 2：上传文件并配置（1分钟）

**操作：**
1. 拖拽上传 Excel 文件到 Judge Only 区域
2. 系统自动解析列名
3. 进行列映射：
   - 问题列 → 选择「问题」
   - 模型列 → 勾选「模型A」「模型B」
4. 输入评测场景：「中文客服对话质量评估」

**讲解词：**
> "上传后系统自动解析，只需简单映射列名，无需复杂的数据预处理。"

#### 步骤 3：AI 生成评测维度（1分钟）

**操作：**
1. 点击「AI 生成维度」按钮
2. 等待 3-5 秒，展示生成结果

**展示生成的维度：**
```json
{
  "dimensions": [
    {"name": "理解准确度", "type": "scale", "weight": 0.3, "description": "是否正确理解用户意图"},
    {"name": "回答完整性", "type": "scale", "weight": 0.3, "description": "信息是否全面无遗漏"},
    {"name": "语气友好度", "type": "scale", "weight": 0.2, "description": "语气是否亲切自然"},
    {"name": "是否解决问题", "type": "binary", "weight": 0.2, "description": "是否直接回答了用户问题"}
  ]
}
```

**讲解词：**
> "这是 EvalMatrix 的核心创新之一 —— AI 自动生成定制评测维度。系统根据场景描述，自动生成匹配的评测维度和权重，支持 scale、binary、categorical 三种打分类型。"

#### 步骤 4：启动评测（30秒）

**操作：**
1. 点击「🚀 Judge Only 评测」按钮
2. 展示 SSE 流式进度条
3. 说明支持中断，已完成结果自动保存

**讲解词：**
> "评测过程实时可见，支持一键中断，已完成的评测结果会自动保留，避免资源浪费。"

**关键截图/录屏位置：**
1. 文件上传和列映射界面
2. AI 生成维度结果展示
3. 评测进度实时推送

---

### 【第 3 幕】量化面板：数据驱动的决策（2 分钟）

**画面：** 评测完成后自动跳转 Dashboard

#### 展示 1：雷达图对比（30秒）

**讲解词：**
> "评测完成后，系统自动生成量化面板。首先是雷达图，直观展示两个模型在各维度的能力对比，优劣势一目了然。"

**展示要点：**
- 模型 A 在「理解准确度」得分高
- 模型 B 在「语气友好度」得分高
- 整体面积对比显示综合能力差异

#### 展示 2：排行榜与统计分析（30秒）

**讲解词：**
> "我们提供三种科学化的计算方法：加权平均、熵权法、PCA 综合排名，消除主观偏差，还原最真实的评价。"

**展示：**
- 切换三种算法，展示排名变化
- 说明熵权法自动计算维度重要性
- PCA 降维生成综合得分

#### 展示 3：详细数据表（30秒）

**讲解词：**
> "明细表展示逐题得分和 Judge 的推理过程，每一条评分都有据可依，可追溯、可审计。"

**展示：**
- 逐题展示问题、模型回答、各维度得分
- 展开查看 Judge 的 Reasoning（推理过程）

#### 展示 4：历史记录（30秒）

**操作：**
1. 切换到「量化面板」页面
2. 点击「刷新列表」加载历史评测
3. 点击任意记录加载完整结果

**讲解词：**
> "所有评测结果自动保存，支持历史回溯和版本对比，建立可复现的质量基线。"

**关键截图位置：**
1. 雷达图对比（最直观的展示）
2. 排行榜和算法切换
3. 明细表和 Reasoning 展示
4. 历史记录列表

### 【第 4 幕】人工复核：人机协同的质量保障（1.5 分钟）

**场景：** 对 Judge 评分进行人工审核和修正

#### 展示 1：盲审模式（30秒）

**操作：**
1. 进入「人工复核」页面
2. 勾选「盲审模式」
3. 展示模型名称被隐藏为「模型 A」「模型 B」

**讲解词：**
> "盲审模式隐藏模型名称，避免品牌偏见，让评测回归质量本身。"

#### 展示 2：逐题审核与修正（1分钟）

**操作：**
1. 展示左右对比布局
2. 点击某维度得分进行编辑
3. 修改分数后点击「同步到 Dashboard」
4. 切换回 Dashboard 展示数据已更新

**讲解词：**
> "人工复核支持逐题对比和分数修正，修改后自动同步到量化面板，实现人机协同的质量保障闭环。"

**关键截图位置：**
1. 人工复核界面（盲审模式）
2. 分数修正操作
3. Dashboard 同步更新

---

### 【第 5 幕】高级功能：复杂场景支持（1.5 分钟）

#### 展示 1：V2 上下文评分（45秒）

**适用场景：** 翻译质量评估、长文本理解

**操作：**
1. 切换 API 版本为 V2
2. 上传文件级上下文（如会议主题文档）
3. 勾选行级附加上下文字段
4. 说明适合需要背景知识的复杂场景

**讲解词：**
> "V2 上下文评分引擎支持文件级上下文和行级附加字段，专为翻译、同传等复杂场景设计，让评测具备上下文理解能力。"

#### 展示 2：数据对齐工具（45秒）

**适用场景：** ASR+翻译评测

**操作：**
1. 进入「数据对齐」页面
2. 上传 XLSX（ASR 结果）和 TXT（原文）
3. 展示自动对齐结果
4. 低置信度段落高亮显示
5. 说明 LLM 辅助精修流程

**讲解词：**
> "针对 ASR 识别结果与原文粒度不一致的问题，我们开发了专门的数据对齐工具，结合字符级匹配和 LLM 精修，实现高精度对齐。"

**关键截图位置：**
1. V2 上下文配置界面
2. 数据对齐结果展示
3. 低置信度段落高亮

## 四、现场演示技巧

### 4.1 时间分配建议

| 环节 | 时间 | 重点 |
|-----|------|------|
| 开场介绍 | 1 min | 产品定位、三大优势 |
| Judge Only 演示 | 3 min | AI 生成维度、实时评测 |
| 量化面板展示 | 2 min | 雷达图、排行榜、Reasoning |
| 人工复核展示 | 1.5 min | 盲审模式、分数修正 |
| 高级功能简述 | 1.5 min | V2 上下文、数据对齐 |
| 总结收尾 | 1 min | 核心价值、未来展望 |

### 4.2 应急预案

| 突发情况 | 应对策略 |
|---------|---------|
| API 调用慢/失败 | 提前准备缓存的历史评测记录，直接加载展示 |
| 文件上传失败 | 准备已上传好的测试项目，直接展示结果页面 |
| 页面加载异常 | 使用截图/录屏作为备用，继续讲解 |
| 时间不够 | 跳过高级功能，重点保证前 3 幕的完整展示 |

### 4.3 讲解要点强调

**必须强调的核心价值：**
1. **AI 生成维度** —— 创新点，体现智能化
2. **科学化计算** —— 熵权法、PCA，体现专业性
3. **端到端自动化** —— 效率提升 100 倍，体现实用价值
4. **人机协同** —— 盲审+复核，体现质量保障

**数据支撑：**
- 与人工评测相关性 0.858
- 效率提升 100 倍
- 已投入真实生产环境

如无法现场演示，可提交录屏视频。

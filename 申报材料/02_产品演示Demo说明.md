## 一、演示环境准备

### 1.1 启动方式

**Windows:**
```cmd
cd D:\POPO\AItester0227\AItester0227 - latest
双击 start.bat
```

**macOS / Linux:**
```bash
cd AItester0227
./start.sh
```

### 1.2 访问地址

- 前端界面：http://localhost:3000
- 后端 API：http://localhost:8000
- API 文档：http://localhost:8000/docs

### 1.3 演示前配置

1. 打开「模型管理」页面
2. 配置 Judge 模型（API Key、Model、URL）
3. 添加被测产品（至少 2 个用于对比）

## 二、演示流程（建议 8-10 分钟）

### 【第 1 幕】产品概览（1 分钟）

**画面：** 登录后主界面

**讲解要点：**
- 左侧导航栏：工作台、评测报告、人工复核、模型管理、数据对齐
- 三种评测模式：Judge Only / Excel Batch / Single Case
- 整体布局简洁，功能分区清晰

**关键截图位置：**
1. 主界面全貌（展示导航栏和工作台）
2. 模型管理页面（展示已配置的产品列表）

---

### 【第 2 幕】核心功能演示 - Judge Only 评测（3 分钟）

**场景：** 已有模型输出，需要批量评分

**操作步骤：**

1. **准备测试数据**
   - 展示 Excel 文件格式：问题列、模型 A 回答列、模型 B 回答列
   - 说明：支持 Excel / JSON / JSONL / ZIP 格式

2. **上传文件并配置**
   - 拖拽上传 Excel 文件
   - 列映射：选择「问题列」「模型列」
   - 输入评测场景：「中文客服对话质量评估」

3. **AI 生成评测维度**
   - 点击「AI 生成维度」按钮
   - 展示生成的维度：
     - 理解准确度 (scale 1-10)
     - 回答完整性 (scale 1-10)
     - 语气友好度 (scale 1-10)
     - 是否解决问题 (binary)

4. **启动评测**
   - 点击「🚀 Judge Only 评测」
   - 展示实时进度条
   - 说明：支持中断，已完成结果自动保存

**关键截图/录屏位置：**
1. 文件上传和列映射界面
2. AI 生成维度结果
3. 评测进度展示

---

### 【第 3 幕】评测报告 Dashboard（2 分钟）

**画面：** 评测完成后自动跳转 Dashboard

**展示内容：**

1. **雷达图对比**
   - 两个模型的多维度能力对比
   - 直观展示各自优劣势

2. **排行榜**
   - 综合得分排名
   - 各维度得分明细

3. **详细数据表**
   - 逐题展示问题和模型回答
   - 各维度得分和 Judge 推理过程

4. **历史记录加载**
   - 切换到「评测报告」页面
   - 展示历史评测列表
   - 点击任意记录加载完整结果

**关键截图位置：**
1. 雷达图对比（最直观的展示）
2. 排行榜和得分明细
3. 历史记录列表

---

### 【第 4 幕】人工复核（2 分钟）

**场景：** 对 Judge 评分进行人工审核和修正

**操作步骤：**

1. **进入人工复核页面**
   - 展示盲审模式（隐藏模型名称）
   - 说明：避免品牌偏见

2. **逐题审核**
   - 展示左右对比布局
   - 模型 A vs 模型 B 的回答对比
   - 各维度得分可编辑

3. **修正分数**
   - 修改某维度得分
   - 点击「同步到 Dashboard」
   - 展示 Dashboard 数据实时更新

**关键截图位置：**
1. 人工复核界面（盲审模式）
2. 分数修正操作

---

### 【第 5 幕】高级功能（2 分钟）

#### 5.1 V2 上下文评分

**适用场景：** 翻译质量评估

**演示要点：**
- 切换 API 版本为 V2
- 上传文件级上下文（如会议主题文档）
- 勾选行级附加上下文字段
- 说明：适合需要背景知识的复杂场景

#### 5.2 ZIP 项目批量评测

**演示要点：**
- 上传包含 manifest.json 的 ZIP 文件
- 展示多文件选择和解析
- 每个文件可配置传入模式（全文/滑窗）
- SSE 流式进度展示

#### 5.3 数据对齐工具

**适用场景：** ASR+翻译评测

**演示要点：**
- 上传 XLSX（ASR 结果）和 TXT（原文）
- 展示自动对齐结果
- 低置信度段落高亮
- 说明 LLM 辅助精修流程

**关键截图位置：**
1. V2 上下文配置界面
2. ZIP 项目解析界面
3. 数据对齐结果展示

## 四、演示技巧

### 4.1 现场演示建议

1. **提前预热** —— 演示前 5 分钟启动服务，确保流畅
2. **准备备份** —— 截图/录屏作为备用，防止现场网络问题
3. **控制节奏** —— 重点展示 Judge Only + Dashboard，其他功能简述
4. **突出亮点** —— 强调 AI 生成维度、雷达图对比、盲审复核

### 4.2 可能的问题及应对

| 问题 | 应对策略 |
|-----|---------|
| API 调用慢 | 提前准备缓存结果，演示时加载历史记录 |
| Judge 模型配置失败 | 展示预配置好的环境，跳过配置步骤 |
| 文件上传失败 | 准备已上传好的测试项目，直接展示结果 |

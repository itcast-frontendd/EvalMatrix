# EvalMatrix —— AI 驱动的产品智能评测系统

> AI 是科技企业的生命线，而科学、高效的产品评测则是创造优秀 AI 产品的基石。

---

## 一、产品名称

**EvalMatrix** —— AI 驱动的产品智能评测系统

**命名寓意：**
- **Eval** = Evaluation（评测）
- **Matrix** = 矩阵化、系统化、多维度的评测体系
- 整体寓意：构建 AI 产品评测的「基座设施」，让评测像矩阵运算一样科学、精确、可复现

---

## 二、产品定位

EvalMatrix 是一套面向 AI 全场景的自动化智能评测平台，通过「AI 评测 AI」的创新范式，解决传统人工评测主观性强、成本高、维度不全、接入复杂等痛点，实现 AI 产品从能力验证、迭代对比到上线验收的全流程自动化评测。

**核心价值主张：**
1. **拒绝「拍脑袋」打分** —— 用数学逻辑还原最真实的评价
2. **通用架构，即插即用** —— 支持任意 OpenAI 兼容 API，零改造成本接入
3. **真正的端到端自动化** —— 把繁琐的评测工作变成一条全自动流水线

**产品形态：** 本地部署的 Web 应用（前后端分离架构），支持 SaaS 云端版本

## 四、解决问题

### 4.1 问题本质

AI 产品的浪潮汹涌，但目前行业更多关注「如何做一个好的产品」，而对「如何评价一个产品的好坏」缺乏足够的思考。评测环节的缺失导致：
- 模型上线后才发现质量问题
- 版本迭代缺乏量化回归验证
- 竞品对比停留在「感觉更好」的主观层面
- 无法建立可复现的质量基线

### 4.2 EvalMatrix 的解决方案

| 痛点 | EvalMatrix 解决方案 | 效果 |
|-----|-------------------|------|
| 主观性强 | AI Judge 自动评分 + 数学建模（熵权法、PCA） | 评测标准统一，结果可复现 |
| 成本高昂 | 端到端自动化流水线，批量并行评测 | 效率提升 100 倍以上 |
| 维度缺失 | AI 自动生成定制维度，支持 scale/binary/categorical 多类型 | 全方位质量画像 |
| 接口复杂 | 调用与测评解耦，支持任意 OpenAI 兼容 API | 零改造成本接入 |

## 五、如何解决（技术方案）

### 5.1 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端层 (Next.js 14)                       │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐     │
│  │  评测工作台  │ │  量化面板   │ │  人工复核   │ │  模型配置   │     │
│  │  (3种模式)  │ │ (雷达/排行) │ │  (盲审模式) │ │ (即插即用) │     │
│  └────────────┘ └────────────┘ └────────────┘ └────────────┘     │
└──────────────────────────┬──────────────────────────────────────┘
                           │ HTTP API
┌──────────────────────────┴──────────────────────────────────────┐
│                        后端层 (FastAPI)                         │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐  │
│  │  评测引擎   │ │  AI Judge  │ │  任务管理   │ │  数学建模   │  │
│  │            │ │   调用     │ │            │ │(熵权法/PCA)│  │
│  └────────────┘ └────────────┘ └────────────┘ └────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

**技术栈：**
- **前端：** Next.js 14 + TypeScript + Tailwind CSS + Recharts（可视化）
- **后端：** FastAPI + Pydantic + Pandas + Scikit-learn（数学建模）
- **AI 引擎：** 支持主流大模型作为 Judge，具备强大的 Prompt 工程优化

### 5.2 核心技术创新

#### 创新点 1：三重评测模式（调用与测评解耦）

| 模式 | 技术实现 | 适用场景 |
|-----|---------|---------|
| **Judge Only（裁判模式）** | 导入已有模型输出，AI Judge 自动评分 | 已有数据，快速评估 |
| **Excel Batch（批量模式）** | 批量调用产品 API → 获取回答 → 自动评分 | 端到端自动化测试 |
| **Single Case（单例模式）** | 实时调用单条 Prompt 即时对比 | 快速调试、演示验证 |

**核心优势：** 调用与测评解耦，支持任意阶段介入，灵活适配不同工作流程。

#### 创新点 2：AI 自动化生成定制评测维度

- 用户输入场景描述（如「医疗问诊对话」）
- 系统调用 LLM 自动生成匹配的评测维度
- 支持三种打分类型：
  - **scale**：1-10 分制（如准确性、流畅度）
  - **binary**：是/否判断（如是否解决问题）
  - **categorical**：分类选择（如回答类型：专业/通俗/敷衍）

#### 创新点 3：科学化计算方法与量化面板

**多维度统计分析：**
- **加权平均**：用户自定义维度权重
- **熵权法**：自动计算维度重要性，消除主观偏差
- **PCA 综合排名**：主成分分析降维，生成综合得分

**可视化面板：**
- 雷达图：多维度能力直观对比
- 排行榜：综合得分排名
- 趋势图：多轮稳定性分析

#### 创新点 4：V2 上下文评分引擎（复杂场景支持）

针对翻译、同传等需要上下文理解的场景：
- **文件级上下文**：传入完整参考文档作为背景
- **行级附加字段**：支持上下文感知评分
- **滑窗算法**：处理超长文本，保持上下文连贯性

#### 创新点 5：数据对齐技术（ASR+翻译场景）

针对 ASR 识别结果与原文逐字稿粒度不一致的问题：
1. 字符级序列匹配（SequenceMatcher）
2. 低置信度段落调用 LLM 精修
3. 自动处理截断、重叠、空缺等问题

## 六、产品功能说明

### 6.1 功能模块总览

```
EvalMatrix
├── 评测工作台
│   ├── Judge Only 裁判模式
│   ├── Excel Batch 批量模式
│   ├── Single Case 单例模式
│   └── ZIP 项目批量评测
├── 量化面板 (Dashboard)
│   ├── 雷达图对比
│   ├── 排行榜
│   ├── 明细数据表
│   └── 历史记录管理
├── 人工复核
│   ├── 盲审模式（隐藏模型名称）
│   ├── 逐题对比
│   └── 分数修正同步
├── 模型管理
│   ├── Judge 模型配置
│   └── 被测产品配置（OpenAI 兼容 / Dify）
└── 数据对齐工具
    ├── XLSX ↔ TXT 对齐
    └── LLM 辅助精修
```

### 6.2 核心功能详解

#### 1. 评测工作台

**Judge Only 模式（最常用）：**
1. 准备数据文件（Excel/JSON/JSONL），包含「问题」和「模型回答」
2. 上传文件，映射列（问题列、模型列、可选图片列/轮次列）
3. 输入评测场景，点击「AI 生成维度」
4. 一键启动评测，SSE 流式实时查看进度
5. 完成后自动跳转 Dashboard

**V2 批量上下文模式：**
- 切换 API 版本为 V2
- 上传文件级上下文（如会议主题文档）
- 勾选行级附加上下文字段
- 适合翻译质量评估等复杂场景

#### 2. 量化面板 Dashboard

- **雷达图**：多维度能力可视化对比，直观展示模型优劣势
- **排行榜**：综合得分排名，支持加权/熵权/PCA 三种算法
- **明细表**：逐题展示问题、模型回答、各维度得分、Judge 推理过程
- **历史记录**：加载任意历史评测，对比分析版本演进

#### 3. 人工复核

- **盲审模式**：隐藏模型名称，避免品牌偏见
- **逐题对比**：左右布局对比多个模型回答
- **自动识别控件**：binary/categorical 维度呈现对应交互控件
- **同步机制**：修改自动回写到原始结果文件，Dashboard 实时更新

#### 4. 评测中断与恢复

- 所有评测模式支持一键中断
- 前端取消 HTTP 请求 + 后端终止剩余任务
- 已完成的部分结果自动保留，避免资源浪费

### 6.3 支持的模型类型

| 类型 | 配置方式 | 示例 |
|-----|---------|------|
| OpenAI 兼容 API | 粘贴代码片段自动解析 | OpenRouter、文心一言、通义千问 |
| Dify Workflow | 填写 API URL 和 Key | Dify 搭建的 Agent 工作流 |
| 本地模型 | 配置本地 API 地址 | vLLM、Ollama、LM Studio |

## 七、用户案例

### 案例 1：同传翻译质量评估（已投入生产）

**背景：** 某翻译团队需要评估 AI 同传系统的翻译质量

**挑战：**
- ASR 识别结果与原文逐字稿粒度不一致
- 需要上下文感知的翻译质量评分
- 批量处理大量会议录音（N = 53,540 条）

**EvalMatrix 解决方案：**
1. 使用数据对齐工具将 ASR 结果与原文对齐
2. 配置 V2 上下文评分，传入会议主题作为背景
3. 设置多维度评分：准确性、流畅性、术语一致性、延迟
4. 批量评测，生成综合报告

**效果验证：**
- **效率提升：** 从人工 2 天缩短到自动 2 小时，提升 100 倍
- **相关性验证：** 与人工评测的相关度达到 **0.858**，证明自动化评测的科学性
- **质量发现：** 发现特定领域的术语翻译问题，指导模型优化

### 案例 2：写作型 AI 产品测评（全链路一条龙）

**场景：** 评估某 AI 写作助手的内容生成质量

**流程：**
1. 配置产品 API（Single Case 模式）
2. 输入评测场景：「科技文章写作」
3. AI 自动生成维度：内容准确性、逻辑连贯性、语言流畅度、原创性
4. 实时调用产品 API 获取回答
5. Judge 模型自动评分，输出各维度得分及总分排名
6. 展示评分 Reasoning（推理过程）

**价值：** 全链路自动化，从输入到报告输出无需人工干预。

### 案例 3：客服机器人效果对比

**背景：** 某电商公司评估 3 个客服机器人方案

**场景：**
- 多轮对话理解
- 商品咨询回答
- 售后问题处理

**EvalMatrix 配置：**
1. 准备 200 条真实客服对话作为测试集
2. 配置 3 个被测产品（不同供应商的机器人）
3. AI 生成评测维度：理解准确度、回答完整性、语气友好度、解决率
4. 运行 3 轮稳定性测试

**结果：**
- 量化对比出最优方案
- 发现某模型在售后场景表现明显落后
- 为采购决策提供数据支撑，避免「拍脑袋」选型

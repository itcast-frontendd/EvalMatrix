# PTSD - Product Test Smart Dog
## AI 驱动的产品智能评测系统

---

## 一、产品名称

**PTSD (Product Test Smart Dog)** —— AI 驱动的产品智能评测系统

> 命名寓意：像训练有素的测试犬一样，敏锐、可靠、不知疲倦地为产品质量把关。

---

## 二、产品定位

PTSD 是一款面向 AI 产品团队的专业评测工具，解决大模型产品上线前缺乏标准化、可量化评估手段的痛点。

**核心价值主张：**
- 让 AI 产品评测从「拍脑袋」变成「用数据说话」
- 支持多模型横向对比、自动化评分、人工复核全流程
- 提供可复现、可追踪的评测历史管理

**产品形态：** 本地部署的 Web 应用（前后端分离架构）

## 三、用户群体

| 用户角色 | 典型场景 | 核心需求 |
|---------|---------|---------|
| **AI 产品经理** | 上线前评估模型效果 | 快速对比多个模型版本，量化决策依据 |
| **算法工程师** | 模型迭代验证 | 批量测试、稳定性分析、回归验证 |
| **质量测试团队** | 日常质量监控 | 标准化评测流程、历史数据追踪 |
| **同传/翻译团队** | 翻译质量评估 | 上下文感知评测、多维度打分 |

## 四、解决什么问题

### 行业痛点

1. **评测标准不统一** —— 不同团队、不同时间评测结果无法对比
2. **人工评测成本高** —— 大量测试用例需要人工逐一检查
3. **多模型对比困难** —— 缺乏工具化手段进行 A/B 测试
4. **评测结果难沉淀** —— 历史数据分散，无法追踪产品演进
5. **复杂场景支持弱** —— 翻译、长文本等场景需要上下文理解

### PTSD 的解决方案

| 痛点 | PTSD 解决方案 |
|-----|--------------|
| 标准不统一 | 支持自定义评测维度 + AI 自动生成维度 |
| 人工成本高 | Judge 模型自动评分 + 人工复核抽样 |
| 对比困难 | 多产品并行测试，雷达图直观对比 |
| 难沉淀 | 自动保存评测历史，支持回溯分析 |
| 复杂场景 | V2 上下文评分、ZIP 项目批量评测 |

```
└─────────────────────────────────────────────────────────────┘
```

### 5.2 核心技术创新

#### 1. 三重评测模式

| 模式 | 技术实现 | 适用场景 |
|-----|---------|---------|
| **Judge Only** | 导入已有模型输出，Judge 自动评分 | 已有数据，快速评估 |
| **Excel Batch** | 批量调用产品 API → 获取回答 → 评分 | 端到端自动化测试 |
| **Single Case** | 实时调用单条 Prompt 对比 | 快速调试、演示 |

#### 2. 智能评测维度生成

- 用户输入场景描述（如「医疗问诊对话」）
- 系统调用 LLM 自动生成匹配的评测维度
- 支持 scale(1-10)、binary(是/否)、categorical(分类) 三种打分类型

#### 3. V2 上下文评分引擎

针对翻译、同传等复杂场景：
- **文件级上下文**：传入完整参考文档
- **行级附加字段**：支持上下文感知评分
- **滑窗算法**：处理超长文本，保持上下文连贯性

#### 4. 多维度统计分析

- **加权平均**：用户自定义维度权重
- **熵权法**：自动计算维度重要性
- **PCA 综合排名**：降维分析，生成综合得分

#### 5. 稳定性分析

支持同一数据集多次评测（Multi-Run）：
- 观察分数趋势和方差
- 识别模型输出的不稳定性
- 生成置信区间报告

### 5.3 数据对齐技术（特色功能）

针对 ASR+翻译评测场景，开发了专门的文本对齐模块：

```
问题：ASR 识别文本与原文逐字稿粒度不一致，且存在识别错误

解决方案：
1. 字符级序列匹配（SequenceMatcher）
2. 低置信度段落调用 LLM 精修
3. 自动处理截断、重叠、空缺等问题
```

## 六、产品功能说明

### 6.1 功能模块总览

```
PTSD
├── 评测工作台
│   ├── Judge Only 模式
│   ├── Excel Batch 模式
│   ├── Single Case 模式
│   └── ZIP 项目批量评测
├── 评测报告 (Dashboard)
│   ├── 雷达图对比
│   ├── 排行榜
│   ├── 明细数据表
│   └── 历史记录管理
├── 人工复核
│   ├── 盲审模式
│   ├── 逐题对比
│   └── 分数修正同步
├── 模型管理
│   ├── Judge 模型配置
│   └── 被测产品配置
└── 数据对齐工具
    ├── XLSX ↔ TXT 对齐
    └── LLM 辅助精修
```

### 6.2 核心功能详解

#### 1. 评测工作台

**Judge Only 模式（最常用）：**
1. 准备数据文件（Excel/JSON/JSONL），包含「问题」和「模型回答」
2. 上传文件，映射列（问题列、模型列、可选图片列/轮次列）
3. 输入评测场景，点击「AI 生成维度」
4. 一键启动评测，实时查看进度
5. 完成后自动跳转 Dashboard

**V2 批量上下文模式：**
- 支持文件级上下文（文本/文件上传）
- 可勾选行级附加上下文字段
- 适合翻译质量评估等复杂场景

**ZIP 项目评测：**
- 打包多文件一次评测
- 支持全文传入 / 滑窗两种上下文模式
- SSE 流式进度推送

#### 2. 评测报告 Dashboard

- **雷达图**：多维度能力可视化对比
- **排行榜**：综合得分排名
- **明细表**：逐题得分、推理过程
- **历史记录**：加载任意历史评测，对比分析

#### 3. 人工复核

- **盲审模式**：隐藏模型名称，避免偏见
- **逐题对比**：拖拽调整答案区域
- **自动识别**：binary/categorical 维度呈现对应控件
- **同步机制**：修改自动回写到原始结果文件

#### 4. 评测中断与恢复

- 所有评测模式支持一键中断
- 前端取消 HTTP 请求 + 后端终止剩余任务
- 已完成的部分结果自动保留

### 6.3 支持的模型类型

| 类型 | 配置方式 | 示例 |
|-----|---------|------|
| OpenAI 兼容 API | 粘贴代码片段自动解析 | OpenRouter、自定义 API |
| Dify Workflow | 填写 API URL 和 Key | Dify 搭建的 Agent |
| 本地模型 | 配置本地 API 地址 | vLLM、Ollama |

## 七、用户案例

### 案例 1：同传翻译质量评估

**背景：** 某翻译团队需要评估 AI 同传系统的翻译质量

**挑战：**
- ASR 识别结果与原文逐字稿粒度不一致
- 需要上下文感知的翻译质量评分
- 批量处理大量会议录音

**PTSD 解决方案：**
1. 使用数据对齐工具将 ASR 结果与原文对齐
2. 配置 V2 上下文评分，传入会议主题作为背景
3. 设置多维度评分：准确性、流畅性、术语一致性、延迟
4. 批量评测 50+ 场会议，生成综合报告

**效果：**
- 评测效率提升 10 倍（从人工 2 天到自动 2 小时）
- 发现特定领域的术语翻译问题，指导模型优化

### 案例 2：客服机器人效果对比

**背景：** 某电商公司评估 3 个客服机器人方案

**场景：**
- 多轮对话理解
- 商品咨询回答
- 售后问题处理

**PTSD 配置：**
1. 准备 200 条真实客服对话作为测试集
2. 配置 3 个被测产品（不同供应商的机器人）
3. AI 生成评测维度：理解准确度、回答完整性、语气友好度、解决率
4. 运行 3 轮稳定性测试

**结果：**
- 量化对比出最优方案
- 发现某模型在售后场景表现明显落后
- 为采购决策提供数据支撑

### 案例 3：内部模型迭代验证

**背景：** AI 团队每周发布模型新版本，需要快速回归测试

**PTSD 工作流：**
1. 建立标准测试集（500 条覆盖各场景）
2. 配置自动化流水线调用 PTSD API
3. 每次发布自动运行评测
4. 对比历史版本，生成趋势报告

**收益：**
- 版本质量可视化，避免「感觉更好」的主观判断
- 快速发现回归问题，及时回滚

## 八、商业化方式与未来前景

### 8.1 商业化路径

| 阶段 | 模式 | 说明 |
|-----|------|------|
| **当前** | 开源/内部工具 | 免费使用，积累用户案例 |
| **短期** | SaaS 订阅 | 提供云端托管版本，按评测量计费 |
| **中期** | 企业版授权 | 私有化部署、定制开发、技术支持 |
| **长期** | 生态平台 | 评测标准市场、模型认证服务 |

### 8.2 目标市场

- **直接客户：** AI 产品团队、算法工程师、测试团队
- **间接客户：** AI 模型厂商（需要评测工具展示产品优势）
- **市场规模：** 随着大模型应用爆发，模型评测需求快速增长

### 8.3 竞争优势

1. **本地化部署** —— 数据不出境，满足企业安全要求
2. **灵活配置** —— 支持任意 OpenAI 兼容 API，不限定模型
3. **上下文感知** —— V2 评分引擎支持复杂场景
4. **开源生态** —— 可二次开发，社区贡献评测维度模板

### 8.4 未来规划

**近期（3 个月）：**
- 完善可视化报告，支持导出 PPT
- 增加更多统计分析方法
- 优化大文件处理性能

**中期（6-12 个月）：**
- 支持多模态评测（图文、视频）
- 引入强化学习自动优化评测维度
- 建立行业评测标准库

**长期（1-2 年）：**
- 构建模型评测认证体系
- 对接模型交易平台，提供质量背书
- 成为 AI 产品质量的基础设施
